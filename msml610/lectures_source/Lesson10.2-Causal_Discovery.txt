# ##############################################################################

## #############################################################################
## Inferring Causal Structure from Observational Data
## #############################################################################

// Papers:
// - 2023, Zanga et al., "A Survey on Causal Discovery: Theory and Practice"
//   https://arxiv.org/pdf/2305.10032
// - 2020, Glymour et al., "Review of Causal Discovery Methods Based on Graphical Models"
//   https://par.nsf.gov/servlets/purl/10125762
// - 2015, Guo et al., "A survey on causal inference"
//   https://qiniu.pattern.swarma.org/attachment/A%20Survey%20on%20Causal%20Inference.pdf
// - 2006, Meek, "Causal inference and causal explanation with background knowledge"
//   https://arxiv.org/pdf/1302.4972

* Causal Discovery: Motivation

- So far we have **assumed a causal DAG was given**
  - E.g., drawn by a domain expert

- **Question**
  - What if we _do not_ know the DAG?
  - Can we learn causal structure directly from data?

- **Causal discovery** (a.k.a. _structure learning_)
  - Recover the causal DAG from observational and optionally interventional data

- **Why it matters**
  - _Observational data_ is plentiful but ambiguous
  - _Experiments_ are expensive, slow, or unethical
    - E.g., cannot randomize exposing humans to asbestos
  - **Goal**: extract causal hypotheses that can be tested later

* Causal Discovery: Definition

- **Causal discovery** is the process of inferring from a dataset sampled from an
  unknown data-generating process:
  $$
  \mathcal{D} \sim \Pr(X_1, \dots, X_n)
  $$
  a graph $G$ (or equivalence class) describing causal relations among
  $X_1, ..., X_n$
  such that:
  $$
  \Pr(X_1, \dots, X_n) = \prod_{i=1}^n \Pr(X_i \mid Pa_G(X_i))
  $$
  where $Pa_G(X_i)$ are the parents of $X_i$ in $G$

- **Example**: 
  - Macroeconomic drivers
    - From time-series data, infer causal links between inflation, unemployment,
      and interest rates
  - Education $\to$ income
    - From observational survey data, estimate whether education causally affects
      earnings (vs confounding)
  - Peer influence in social networks
    - From user behavior data, infer whether friends influence each other's
      purchases or opinions

* Observational vs. Interventional Data

\begingroup
\footnotesize
|                  | **Observational Data**                   | **Interventional Data**                 |
|------------------|------------------------------------------|-----------------------------------------|
| **Definition**   | Data collected by *watching* the system  | Data collected under *do*-interventions |
| **Manipulation** | No variable is manipulated               | Variables are actively manipulated      |
| **Example**      | Hospital records of patients             | Randomized Controlled Trial (RCT)       |
| **Pros**         | Abundant and cheap                       | Identifies causal direction             |
|                  | No ethical constraints                   | Breaks spurious correlations            |
| **Cons**         | Cannot distinguish cause vs correlation  | Expensive, slow, often infeasible       |
|                  | Hidden confounders                       | Ethical issues (e.g., smoking trials)   |
\endgroup

- Discovery algorithms combine both when possible

## #############################################################################
## Identifiability and Causal Sufficiency
## #############################################################################

// Papers:
// - 2012, Maathuis et al., "Estimating high-dimensional intervention effects from observational data"
// - 2010, Kalisch et al., "Causal inference using invariant prediction"
// - 2009, Zhang et al., "On the identifiability of the post-nonlinear causal model"
//   https://arxiv.org/pdf/1205.2599

* Key Assumptions for Causal Discovery

- Causal discovery rests on three main assumptions:
  1. **Causal Markov condition**
     - Each variable is independent of its non-descendants given its parents
     - Equivalent to the factorization $\Pr(X) = \prod_i \Pr(X_i \mid Pa(X_i))$
  2. **Faithfulness**
     - All conditional independencies in the data come from the DAG structure,
       not from _numerical coincidences_ (no cancelling paths)
  3. **Causal sufficiency**
     - No hidden common causes (unobserved confounders)
     - All relevant variables are measured

- Violating any of these can lead to an **incorrect** recovered DAG

* Markov Equivalence Classes

- Can observational data distinguish every DAG?
  - **No**
  - Different DAGs may encode the same conditional independencies

- **Markov equivalent DAGs** share the same:
  - Skeleton (undirected edges)
  - v-structures (colliders $X \to Z \leftarrow Y$)

- The best an observational-only algorithm can return is a **CPDAG**
  (Completed Partially Directed Acyclic Graph) that represents the equivalence
  class

* Identifiability: When Can We Recover the DAG?

- **Identifiable from observational data alone**
  - Only the _Markov equivalence class_ (a CPDAG)
  - Directions of some edges remain ambiguous

- **Fully identifiable under stronger assumptions**
  - _Linear non-Gaussian noise_ (LiNGAM): direction recovered uniquely
  - _Additive noise models_ with nonlinear functions
  - _Post-nonlinear_ models (Zhang et al., 2009)

- **Identifiable with interventional data**
  - Each intervention cuts incoming edges to the target variable
  - Enough interventions recover the full DAG

- **Rule of thumb**: more structure (non-Gaussianity, nonlinearity,
  interventions) $\Rightarrow$ more identifiability

* Faithfulness: A Subtle Assumption

- **Faithfulness**: the observed distribution has _no extra_ independencies
  beyond those implied by the DAG

// TODO(gp): Add diagram
- **Example of violation**
  - Path cancellation
  - Suppose $X \to Y$ and $X \to Z \to Y$
  - If the two paths' effects are equal in magnitude and opposite in sign,
    $X \perp Y$ in the data even though $X$ causes $Y$
  - Algorithm would conclude _no edge_

- In practice, exact cancellations are _measure zero_, but near-cancellations
  in finite samples cause errors

- Discovery is:
  - Statistically correct in the limit
  - Fragile with small data

## #############################################################################
## Practical Limitations of Automated Discovery
## #############################################################################

// Papers:
// - 2018, Loh et al., "Causal discovery with identifiable nonlinear models"
// - 2017, Uhler et al., "Geometries of faithfulness in graphical models"
//   https://arxiv.org/abs/1608.00191

* Practical Limitations

- **Sample complexity**
  - Conditional independence tests need many samples
  - Error rate scales with number of tests: $O(n^2)$ pairs of variables

- **Hidden confounders**
  - Break causal sufficiency
  - Produce spurious edges or wrong directions

- **Selection bias**
  - Sampling mechanism depends on variables of interest
  - E.g., only patients sick enough to visit hospital are measured

- **Finite-sample faithfulness**
  - Weak dependencies look like independencies
  - Statistical power drops rapidly with conditioning set size

* Practical Limitations (cont.)

- **Temporal information is usually ignored**
  - Yet time is the strongest causal prior
  - Some algorithms (Granger-causality, DYNOTEARS) exploit it explicitly

- **Non-stationarity and heterogeneity**
  - Distribution shifts across subpopulations break i.i.d. assumptions

- **Computational complexity**
  - Search space grows _super-exponentially_ in the number of variables $n$:
    $$
    |\{\text{DAGs over } n \text{ nodes}\}| \approx 2^{O(n^2)}
    $$
  - PC algorithm: $O(n^q)$ where $q$ is max degree
  - NP-hard in general for score-based search

- **Bottom line**: discovery is a _hypothesis generator_, not a truth oracle

# When to Use Discovery vs. Domain Knowledge
# ##############################################################################

// Papers:
// - 2021, Gamella et al., "Active learning of linear causal models"
//   https://arxiv.org/abs/2006.05690
// - 2019, Wang et al., "RL-based structural equation modeling"
// - 2013, Mani et al., "Learning the structure of causal models with experimental and observational data"

## #############################################################################
## Discovery as Hypothesis Generation
## #############################################################################

* Discovery vs. Domain Knowledge

- When should we _run an algorithm_ vs _ask an expert_?

\begingroup
\footnotesize
| **Situation**                          | **Recommended approach**                    |
| -------------------------------------- | ------------------------------------------- |
| Well-studied domain (physics, biology) | Expert-drawn DAG, verified by data          |
| Exploratory science (genomics)         | Discovery algorithm as hypothesis generator |
| Policy / medicine (high stakes)        | Expert DAG + RCTs for key edges             |
| Large systems (hundreds of variables)  | Discovery to prune, expert to refine        |
| Clear temporal order                   | Time-based + discovery                      |
\endgroup

- **Key takeaway**: discovery and expertise are _complements_, not substitutes

* Discovery as Hypothesis Generation

- **Discovery algorithms output *candidate* DAGs, not truth**
  - Output depends on assumptions that may not hold
  - Multiple DAGs in the equivalence class are equally compatible

- Use discovery to:
  - Generate **plausible structures** to examine
  - Highlight **surprising edges** worth investigating
  - Flag **missing confounders** (variables with many high-correlation edges)

- Do _not_ use discovery to:
  - Make final policy decisions without expert review
  - Replace RCTs for treatment-effect estimation
  - Claim causation from purely observational pipelines

- **Example**: Sachs et al. (2005) reconstructed a known protein-signaling
  network from single-cell data $\rightarrow$ validated discovery as a
  _scientific_ tool

## #############################################################################
## Combining Discovery with Domain Knowledge
## #############################################################################

// Papers:
// - 2019, Solus et al., "Causal discovery with unknown interventions"
// - 2018, Squires et al., "Active structure learning of causal DAGs"
//   https://arxiv.org/abs/2011.00641
// - 2015, Sachs et al., "Causal protein-signaling networks derived from multiparameter single-cell data"

* Using Domain Knowledge as Constraints

- Partial domain knowledge can **drastically** reduce the search space

- Common forms of prior knowledge:
  - **Forbidden edges**: "variable $X$ cannot cause variable $Y$"
  - **Required edges**: "we know $X \to Y$ from physics"
  - **Tiered ordering**: variables in tier $k$ cannot cause tier $<k$
    - E.g., _demographics $\to$ behavior $\to$ outcome_
  - **Temporal order**: past cannot depend on future

- These constraints are passed to the discovery algorithm as a **"knowledge"
  object** (Meek, 2006)

- **Pros**: faster, more accurate, more interpretable
- **Cons**: wrong priors inject wrong edges

* Combining Automated Discovery with Expert Judgment

- **Typical workflow**:
  1. Expert provides _variable list_ and _temporal ordering_
  2. Expert marks _forbidden_ and _required_ edges
  3. Algorithm proposes a CPDAG over remaining edges
  4. Expert reviews output; refutes edges via subject-matter reasoning
  5. Active learning suggests _next intervention_ to disambiguate edges

- **Example**: _Active causal structure learning_ (Squires et al., 2018)
  - Pick the intervention that _most reduces_ the number of DAGs consistent
    with the data
  - Analogous to active learning in classification

## #############################################################################
## Constraint-Based Methods: PC and FCI
## #############################################################################

// Papers:
// - 1993, Spirtes et al., "An algorithm for fast recovery of sparse causal graphs"
// - 2012, Maathuis et al., "Estimating high-dimensional intervention effects from observational data"
// - 1999, Zhang, "On the completeness of orientation rules for causal discovery in the presence of latent confounders and selection bias"

* PC Algorithm: Idea

- Named after inventors **P**eter Spirtes and **C**lark Glymour (1993)

- **Key idea**: use conditional independence (CI) tests to recover the skeleton
  of the DAG, then orient edges

- **Input**: dataset $\mathcal{D}$, CI test (e.g., partial correlation,
  $G^2$-test)
- **Output**: a CPDAG representing the Markov equivalence class

- **Assumes**:
  - Causal sufficiency (no hidden confounders)
  - Faithfulness
  - Correct CI tests

// - **Implementation**:
// - Use `gCastle.pc()` to run PC algorithm on your data
// - Use `DoWhy` to test conditional independence and refute discovered edges
// - Visualize output DAGs using `CausalNex` for graph operations

* PC Algorithm: Steps

- **Steps**:
  1. Start from a fully connected undirected graph over $X_1, \dots, X_n$
  2. For each pair $(X_i, X_j)$, test $X_i \perp X_j \mid S$ for subsets $S$
     of increasing size
     - If independence found, remove the edge and record the _separating set_
       $S_{ij}$
  3. Identify **v-structures**: for any unshielded triple $X_i - X_k - X_j$, if
     $X_k \notin S_{ij}$ then orient as $X_i \to X_k \leftarrow X_j$
  4. Apply **orientation rules** (Meek) to propagate directions without
     creating new v-structures or cycles

- **Complexity**: $O(n^q)$ in the number of CI tests, where $q$ is the maximum
  degree of the true DAG

* PC Algorithm: Example

::: columns
:::: {.column width=55%}
- **Variables**: Smoking ($S$), Tar ($T$), Cancer ($C$)

- **CI tests**:
  - $S \not\perp C$
  - $S \not\perp T$
  - $T \not\perp C$
  - $S \perp C \mid T$ $\Rightarrow$ separating set $\{T\}$

- **Conclusion**:
  - Remove edge $S - C$
  - No v-structure ($T$ is in separating set)
  - Skeleton: $S - T - C$

* FCI: Dealing with Latent Confounders

- **PC fails** under hidden confounders (causal sufficiency violated)

- **FCI (Fast Causal Inference)** (Spirtes et al., 1995; Zhang, 1999)
  - Extends PC to handle _unobserved_ common causes and selection bias
  - Output: a **PAG** (Partial Ancestral Graph), with edge marks:
    - $\to$: causes
    - $\leftrightarrow$: confounded (unobserved common cause)
    - $\circ\!\to$: ambiguous
    - $\circ\!\!-\!\!\circ$: undetermined

- **Pros**: sound under latent confounders
- **Cons**: slower; output less informative (many $\circ$ marks)

- **Example**: in econometrics with unobserved "ability" confounding
  education and wages, FCI correctly marks $\text{Education}
  \leftrightarrow \text{Wage}$ instead of asserting causation

## #############################################################################
## Score-Based Methods: GES and NOTEARS
## #############################################################################

// Papers:
// - 2023, Ke et al., "Learning sparse causal models is not NP-hard"
//   https://arxiv.org/abs/1309.6824
// - 2018, Zheng et al., "DAGs with NO TEARS: continuous optimization for learning Bayesian networks"
//   https://arxiv.org/abs/1803.01422
// - 2006, Chickering, "Learning equivalence classes of Bayesian network structures"
//   https://arxiv.org/abs/1302.3566

* Score-Based Approach: Idea

- Treat discovery as a **search problem**:
  - Space: all DAGs $G$ over $n$ nodes
  - Objective: score $S(G, \mathcal{D})$ measuring how well $G$ explains the
    data

- **Common scores**:
  - **BIC** (Bayesian Information Criterion):
    $$
    S(G) = \log \Pr(\mathcal{D} \mid G, \hat{\theta}) - \frac{k}{2}\log N
    $$
    where $k$ = number of parameters, $N$ = sample size
  - **BDeu** (Bayesian Dirichlet equivalent uniform)

- **Pros**: sound theory, decomposes over nodes
- **Cons**: DAG space is super-exponential $\rightarrow$ heuristic search

* GES: Greedy Equivalence Search

- **GES** (Chickering, 2002)

- **Input**: dataset $\mathcal{D}$, decomposable score $S(\cdot)$
- **Output**: a CPDAG (equivalence class)

- **Steps**:
  1. **Forward phase**: start from empty graph; add the edge that most
     _increases_ the score, repeat until no improvement
  2. **Backward phase**: remove the edge that most _increases_ the score, until
     no improvement

- Proven to recover the true equivalence class in the **large-sample limit**

- **Complexity**: polynomial per step, but many steps; practical up to
  $n \approx 100$ variables

* NOTEARS: Continuous Optimization

- **Key idea** (Zheng et al., 2018): turn discrete DAG search into a
  **continuous optimization** problem

- Represent the graph by a weighted adjacency matrix $W \in \mathbb{R}^{n
  \times n}$

- DAG-ness constraint encoded as a smooth function:
  $$
  h(W) = \operatorname{tr}(e^{W \odot W}) - n = 0
  $$
  where $h(W) = 0$ iff $W$ corresponds to a DAG

- **Problem**:
  $$
  \min_W \; \|X - X W\|_F^2 + \lambda \|W\|_1 \quad \text{s.t.} \quad h(W) = 0
  $$

- **Pros**: solvable with gradient descent; scales to 100s of variables
- **Cons**: sensitive to data scaling (Reisach et al., 2021); local minima

## #############################################################################
## Non-Gaussian and Functional Methods
## #############################################################################

// Papers:
// - 2006, Shimizu et al., "A linear non-Gaussian acyclic model for causal discovery"
//   https://www.jmlr.org/papers/volume7/shimizu06a/shimizu06a.pdf
// - 2011, Hyvarinen et al., LiNGAM extensions
// - 2021, Reisach et al., "Beware of the Simulated DAG!"
//   https://arxiv.org/abs/2102.13647

* LiNGAM: Linear Non-Gaussian Acyclic Model

- **Assumption**: data are generated by
  $$
  X_i = \sum_{j} b_{ij} X_j + e_i
  $$
  where
  - $b_{ij}$ encodes a DAG (acyclic $B$)
  - Noises $e_i$ are _mutually independent_ and _non-Gaussian_

- **Key insight**: _non-Gaussian_ noise breaks the symmetry
  - Gaussian case: $X \to Y$ and $Y \to X$ are indistinguishable
  - Non-Gaussian: asymmetry $\Rightarrow$ direction is **identifiable**

- Algorithm uses **Independent Component Analysis (ICA)** to recover $B$

- **Example**: in a stock-returns panel, non-Gaussian "fat tails" allow LiNGAM
  to orient edges that Gaussian methods cannot

* Discovery Algorithm Families: Summary

\begingroup \scriptsize
| **Family**       | **Example**   | **Uses**                    | **Output** | **Key limit**                  |
| ---------------- | ------------- | --------------------------- | ---------- | ------------------------------ |
| Constraint-based | PC, FCI       | CI tests                    | CPDAG/PAG  | Sample size for CI tests       |
| Score-based      | GES, NOTEARS  | Likelihood + regularization | CPDAG / W  | Search / optimization hardness |
| Functional       | LiNGAM, ANM   | Non-Gaussianity, nonlinearity | Full DAG | Strong functional assumptions  |
| Interventional   | IGSP, JCI     | Data from interventions     | Full DAG   | Cost of experiments            |
\endgroup

- **Recommendation**:
  - Start with **PC** or **GES** for exploratory analysis
  - Use **LiNGAM** when non-Gaussianity is plausible
  - Use **FCI** when latent confounders are likely
  - Use **interventional** methods when experiments are affordable

## #############################################################################
## Why Standard ML Fails at Causal Discovery
## #############################################################################

// Papers:
// - 2019, Scholkopf, "Causality for Machine Learning"
//   https://arxiv.org/abs/1911.10500
// - 2011, Peters et al., "Causal discovery with continuous additive noise models"
//   https://arxiv.org/abs/1309.6779

* Why Standard ML Fails at Causal Discovery

- Standard ML optimizes **predictive accuracy**, not causal structure
  - Correlation suffices for prediction
  - Direction of arrows is irrelevant to loss

- **Symptoms of the mismatch**:
  - Models exploit _shortcuts_ (spurious features)
  - Fail under distribution shift
  - Cannot answer _"what if we intervene?"_ queries

- **Example**: a classifier trained to detect cows may rely on grass
  backgrounds rather than cow features
  - High test accuracy, zero causal understanding
  - Fails on cows at the beach

- **Scholkopf (2019)**: _"Causality for Machine Learning"_ argues that to build
  robust AI, we must reason about _causal_ mechanisms, not just correlations

* Standard ML vs. Causal Discovery

::: columns
:::: {.column width=50%}
**Standard ML**

- Goal: minimize prediction error
- Assumes i.i.d. data
- Optimizes $\Pr(Y \mid X)$
- Blind to arrow direction
- Fails under shift

- **Pros**
  - Simple, scalable
  - Mature tooling

- **Cons**
  - No interventions
  - No counterfactuals
::::
:::: {.column width=50%}
**Causal Discovery**

- Goal: recover $G$
- Assumes structural model
- Targets $\Pr(Y \mid do(X))$
- Edge directions matter
- Stable under shift

- **Pros**
  - Supports intervention
  - Interpretable

- **Cons**
  - Assumption-heavy
  - Harder to validate

## #############################################################################
## Multiple Models Produce Identical Predictions
## #############################################################################

// Papers:
// - 2013, Margaritis et al., "Learning Bayesian network model structure from data"
// - 2000, Spirtes et al., "Causation, Prediction, and Search"

* The Identifiability Ceiling

- **Observation**: many DAGs induce the **same joint distribution**
  - They are _predictively equivalent_
  - They are _causally_ different

- Three classic patterns:
  1. **Chain**: $X \to Y \to Z$
  2. **Reverse chain**: $Z \to Y \to X$
  3. **Common cause**: $X \leftarrow Y \rightarrow Z$

- All imply $X \perp Z \mid Y$ and are **indistinguishable** from observational
  data

- **Implication**: no matter how much data you collect, you cannot identify
  which of these holds without either:
  - An intervention
  - A functional assumption (e.g., non-Gaussianity)

* Markov Equivalence: Example

:::: {.column width=33%}
**Chain 1**

**Chain 2**
```graphviz

    rankdir=LR;
    node [shape=circle, style="filled", fillcolor="#A6C8F4"];
```

**Common cause**

- All three imply $X \perp Z \mid Y$
- Observational distribution identical
- **Causal implications are very different**:
  - Intervening on $X$ in Chain 1 affects $Z$
  - Intervening on $X$ in Chain 2 does _not_ affect $Z$
  - Intervening on $X$ in Common Cause does _not_ affect $Z$

## Domain Expert Review and Refutation Testing
## #############################################################################

// Papers:
// - 2021, Gamella et al., "Active learning of linear causal models"
//   https://arxiv.org/abs/2006.05690
// - 2018, Squires et al., "Active structure learning of causal DAGs"
//   https://arxiv.org/abs/2011.00641
// - 2008, Castelo et al., "Structural, syntactic, and statistical issues in learning Bayesian networks from data"

* Validating a Discovered DAG

- **A recovered DAG is a *hypothesis*** until validated

- **Validation strategies**:
  1. **Expert review**: do edges match known mechanisms?
  2. **Refutation tests**: does data contradict implied conditional
     independencies?
  3. **Sensitivity analysis**: do results change under perturbations of the
     assumptions?
  4. **Interventional check**: do targeted experiments confirm predicted
     effects?

- **Tools**: `causalnex`, `dowhy`, `CausalDiscoveryToolbox`, `gCastle`

* Refutation Testing

- Given a discovered DAG $G$, check each implied independence against data

- **Procedure**:
  1. Enumerate CI statements implied by $G$ (via d-separation)
  2. Test each on held-out data using an appropriate CI test
  3. Count violations; low count $\Rightarrow$ DAG is _consistent_

- **Placebo tests** (Pearl): introduce a known-null edge; discovery should
  assign it low weight

- **Example** (DoWhy library):
  - Replace treatment with a random variable; effect should vanish
  - Add an unobserved confounder; estimate sensitivity of conclusions

- **Q**: what if the DAG passes all refutation tests?
  - **Answer**: it is _consistent_, not _proven_. Causality is never
    established by data alone; always requires domain-informed interpretation

## #############################################################################
## When Discovery Should Change Your DAG
## #############################################################################

// Papers:
// - 2017, Peters et al., "Elements of Causal Inference"
// - 2018, Squires et al., "Permutation-based causal structure learning with unknown interventions"

* When Discovery Should Change Your DAG

- **Update your DAG when**:
  - Multiple independent algorithms converge on the same edge
  - The new edge is _mechanistically plausible_
  - Refutation tests fail your current DAG
  - Interventional experiments confirm the discovered structure

- **Do *not* update your DAG when**:
  - Edge contradicts well-established mechanism (e.g., time order, physics)
  - Result hinges on weak faithfulness near independence boundary
  - Sample size is too small for reliable CI tests
  - Hidden confounders could explain the edge

- **Rule of thumb**: discovery shifts the _prior probability_ of an edge; it
  does not provide a _posterior certainty_

* Causal Discovery: Key Takeaways

- **Causal discovery** infers DAG structure from data, but only under strong
  assumptions (Markov, faithfulness, sufficiency)

- **Identifiability is limited**
  - Observational data recover only the Markov equivalence class
  - Interventions or functional assumptions are needed for full identification

- **Three algorithm families**:
  - Constraint-based (PC, FCI)
  - Score-based (GES, NOTEARS)
  - Functional / non-Gaussian (LiNGAM)

- **Discovery $\neq$ truth**
  - Use it to _generate_ hypotheses
  - Combine with domain knowledge and interventions
  - Validate via refutation and expert review

- **Bottom line**: automated discovery is a powerful exploratory tool, not a
  replacement for scientific reasoning
