## #############################################################################
## Inferring Causal Structure from Observational Data
## #############################################################################

// Papers:
// - 2023, Zanga et al., "A Survey on Causal Discovery: Theory and Practice"
//   https://arxiv.org/pdf/2305.10032
// - 2020, Glymour et al., "Review of Causal Discovery Methods Based on Graphical Models"
//   https://par.nsf.gov/servlets/purl/10125762
// - 2015, Guo et al., "A survey on causal inference"
//   https://qiniu.pattern.swarma.org/attachment/A%20Survey%20on%20Causal%20Inference.pdf
// - 2006, Meek, "Causal inference and causal explanation with background knowledge"
//   https://arxiv.org/pdf/1302.4972

* Causal Discovery: Motivation

- **Problem**: So far we have assumed the causal DAG was known
  - E.g., given by domain experts

- **Questions**:
  - How do we discover the DAG when it is not specified?
  - Can we learn causal structure directly from observational data?

- **Why it matters**
  - Observational data is plentiful but ambiguous
    - E.g., due to confounding
  - Experiments are expensive, slow, or unethical
    - E.g., cannot randomize exposing humans to asbestos

- **Solution**: _Causal discovery_ (aka structure learning)
  - Recover the causal DAG from observational and/or interventional data
  - Extract causal hypotheses that can be tested and refined

* Causal Discovery: Formal Definition

- **Definition**: Given a dataset $\mathcal{D} \sim \Pr(X_1, \dots, X_n)$,
  _causal discovery_ is the process of inferring a causal DAG $G$ (or
  equivalence class) such that:
  $$\Pr(X_1, \dots, X_n) = \prod_{i=1}^n \Pr(X_i | Pa_G(X_i))$$
  where $Pa_G(X_i)$ are the parents of $X_i$ in $G$

- **Examples**:
  - Macroeconomic drivers: from time-series data, infer causal links between
    inflation, unemployment, and interest rates
  - Education and income: estimate whether education causally affects earnings
    (vs pure confounding)
  - Social networks: infer whether friends influence each other's purchases
    or opinions

* Observational vs Interventional Data

\begingroup
\scriptsize
| **Aspect**   | **Observational Data** | **Interventional Data** |
|--------------|------------------------|------------------------|
| Definition   | Watch the system       | Apply do-interventions |
| Manipulation | Only passive observation | Active variable control |
| Example      | Hospital records       | Randomized Controlled Trial (RCT) |
| Pros         | Abundant, cheap, ethical | Identifies causal direction; breaks spurious correlations |
| Cons         | Cannot distinguish cause from correlation | Expensive, slow, often infeasible |
\endgroup

- **Key insight**: discovery algorithms combine both types when available

## #############################################################################
## Identifiability and Causal Sufficiency
## #############################################################################

// Papers:
// - 2012, Maathuis et al., "Estimating high-dimensional intervention effects from observational data"
// - 2010, Kalisch et al., "Causal inference using invariant prediction"
// - 2009, Zhang et al., "On the identifiability of the post-nonlinear causal model"
//   https://arxiv.org/pdf/1205.2599

* Markov Equivalence

- **Question**: Can observational data distinguish every Directed Acyclic Graph
  (DAG)?
  - No, it cannot
  - Different DAGs can encode identical conditional independencies, forming an
    equivalence class

- **Definition**: _CPDAG_ (Completed Partially Directed Acyclic Graph)
  represents a Markov equivalence class of Bayesian network DAGs
  - A CPDAG includes both directed and undirected edges to capture the
    equivalence class

- **Markov equivalent DAGs** share the same:
  - Skeleton (undirected edges between nodes)
  - v-structures (e.g., $X \to Z \leftarrow Y$)
  - E.g., consider two DAGs, $A \to B \to C$ and $A \leftarrow B \to C$. Both
    have the same skeleton and no v-structures, hence they are Markov equivalent

* Markov Equivalence: Examples

- Chain: $X \to Y \to Z$
  - _"If we intervene on X, Z changes"_

- Reverse chain: $X \leftarrow Y \leftarrow Z$
  - _"Intervening on X has no effect on Z"_

- Confounder: $X \leftarrow Y \to Z$
  - _"Intervening on X has no effect on Z"_

- Same observational correlation, different causal implications
  - All imply $X \perp Z | Y$

- No amount of data can distinguish these structures
  - Without intervention data or additional assumptions, we cannot distinguish
    these structures from correlation alone

* Typical Assumptions for Causal Discovery

- When performing causal discovery, certain assumptions are made:

  - **Causal Markov condition**: each variable is independent of its
    non-descendants given its parents
    - Equivalent to: $\Pr(X) = \prod_i \Pr(X_i | Pa(X_i))$

  - **Faithfulness**: all conditional independencies in the data stem from DAG
    structure, not from numerical coincidences

  - **Causal sufficiency**: no hidden common causes (confounders)
    - All relevant variables are measured

- Violating any assumption leads to incorrect recovered DAGs

* Identifiability

- From observational data alone:
  - Only the Markov equivalence class (CPDAG) can be identified
  - Edge directions remain ambiguous without functional assumptions

- With stronger functional assumptions, full DAG recovery is
  possible
  - Linear non-Gaussian noise (LiNGAM): asymmetry in noise breaks direction
    ambiguity
  - Additive noise models with nonlinear functions
  - Post-nonlinear causal models

- With interventional data, each intervention cuts incoming edges
  - Sufficient interventions can recover the full directed DAG

- **Rule of Thumb**: More structural information leads to more identifiability

* Faithfulness

- **Faithfulness assumption**:
  - Independent variables in data should be independent in causal graph, and vice
    versa
  - The statistical relationships are assumed to "faithfully" reflect the graph
    structure

- **Problem**: path cancellation leads to violation of faithfulness

::: columns
:::: {.column width=60%}

- **Example**:
  - Two paths: $X \to Y$ and $X \to Z \to Y$
  - Equal but opposite effects cancel
  - Algorithm incorrectly concludes: no edge between $X$ and $Y$
    (i.e., $X \perp Y$ from data) though $X$ causes $Y$

## #############################################################################
## Practical Limitations
## #############################################################################

// Papers:
// - 2018, Loh et al., "Causal discovery with identifiable nonlinear models"
// - 2017, Uhler et al., "Geometries of faithfulness in graphical models"
//   https://arxiv.org/abs/1608.00191

* Practical Challenges for Causal Discovery

- **Sample complexity**:
  - Causal discovery from data is sample-intensive
  - Testing all pairs of variables is $O(n^2)$

- **Hidden confounders**:
  - Algorithms assume causal sufficiency (all confounders are measured), rarely
    true
  - Unmeasured variables break core assumptions
  - Create spurious edges (false associations) in the learned graph or incorrect
    edge directions

- **Selection bias**:
  - Sampling mechanism depends on studied variables, creating artificial
    associations
  - E.g., hospitals see sicker patients (selection bias toward severe cases),
    so treatment and outcome appear more associated than in the general
    population

- **Finite-sample faithfulness**:
  - Weak dependencies appear as independencies
  - Statistical power drops with larger conditioning sets
  - "Absence of evidence" seen as "evidence of absence"

* Practical Challenges for Causal Discovery

- **Temporal information often ignored**
  - Time is one of the strongest causal signals, but most discovery algorithms
    ignore temporal ordering
  - Some algorithms (e.g., Granger-causality) exploit temporal order

- **Non-stationarity and heterogeneity**
  - Real-world data rarely satisfies the i.i.d. assumption across populations and
    time
  - Non-stationarity: statistical properties change over time
  - Heterogeneity: different subpopulations have different causal structures
  - Both violate the foundational assumptions most discovery algorithms rely on

- **Computational complexity**:
  - DAG space grows super-exponentially
    $$|\{\text{DAGs over } n \text{ nodes}\}| \approx 2^{O(n^2)}$$
  - PC algorithm: $O(n^q)$ where $q$ is maximum graph degree
  - Score-based search is NP-hard in general
  - Practical implication: exact discovery is infeasible beyond small graphs

## #############################################################################
## Discovery as Hypothesis Generation
## #############################################################################

* Discovery vs Domain Knowledge

- **Question**: When should we _run an algorithm_ vs _ask an expert_?

\begingroup
\footnotesize
| **Situation**                          | **Recommended approach**                    |
| -------------------------------------- | ------------------------------------------- |
| Well-studied domain (physics, biology) | Expert-drawn DAG, verified by data          |
| Exploratory science (genomics)         | Discovery algorithm as hypothesis generator |
| Policy / medicine (high stakes)        | Expert DAG + RCTs for key edges             |
| Large systems (hundreds of variables)  | Discovery to prune, expert to refine        |
| Clear temporal order                   | Time-based + discovery                      |
\endgroup

- **Key takeaway**: discovery and expertise are _complements_, not substitutes

* Discovery as Hypothesis Generation

- Discovery algorithms are a hypothesis generator, not a truth finder
  - Output depends on assumptions that may not hold
  - Multiple DAGs in the equivalence class are equally compatible

- Use discovery to:
  - Generate _plausible structures_ to examine
  - Highlight _surprising edges_ worth investigating
  - Flag _missing confounders_ (variables with many high-correlation edges)

- Do _not_ use discovery to:
  - Make final policy decisions without expert review
  - Replace RCTs for treatment-effect estimation
  - Claim causation from purely observational pipelines

## #############################################################################
## Combining Discovery with Domain Knowledge
## #############################################################################

// Papers:
// - 2019, Solus et al., "Causal discovery with unknown interventions"
// - 2018, Squires et al., "Active structure learning of causal DAGs"
//   https://arxiv.org/abs/2011.00641
// - 2015, Sachs et al., "Causal protein-signaling networks derived from multiparameter single-cell data"
//  (Meek, 2006)

* Using Domain Knowledge as Constraints

- Partial domain knowledge can _drastically_ reduce the search space
  for causal discovery algorithms

- Common forms of prior knowledge:
  - Forbidden edges: "variable $X$ cannot cause variable $Y$"
    - E.g., _"age cannot be caused by purchasing behavior"_
  - Required edges: "we know $X \to Y$ from physics"
  - Tiered ordering:
    - Organize variables into causal levels where lower tiers cannot affect
      higher ones
    - E.g., _demographics $\to$ behavior $\to$ outcome_
  - Temporal order: past cannot depend on future

- These constraints are passed to the discovery algorithm as a "knowledge" object

- **Pros**: faster, more accurate, more interpretable
- **Cons**: wrong priors inject wrong edges

* Combining Automated Discovery with Expert Judgment

- **Typical workflow**:
  1. Expert provides _variable list_ and _temporal ordering_
  2. Expert marks _forbidden_ and _required_ edges
  3. Algorithm proposes a CPDAG over remaining edges
  4. Expert reviews output; refutes edges via subject-matter reasoning
  5. Active learning suggests _next intervention_ that _most reduces_ the number
     of DAGs consistent with the data

// (Squires et al., 2018)

* Metrics
- **Motivation**:
  - Causal discovery algorithms estimate graphs
  - Quantitative metrics assess match with true structure
  - Different errors have different consequences

- **Three Categories of Metrics**:
  - _Edge-level metrics_: TPR (recall), FDR (false discovery rate), F1 score
  - _Structure-level metrics_: Structural Hamming Distance (SHD)
  - _Complexity metrics_: Number of Non-Zero edges (NNZ)

- **Key Distinction**: Structure vs Parameters
  - Structure learning: Right edges and directions?
  - Parameter learning: Correct edge weights?

* Edge-Level Metrics
- **True Positive Rate (TPR)**:
  $\text{TPR} = \frac{\text{TP}}{\text{TP} + \text{FN}}$ = fraction of true
  edges recovered
  - High TPR: Find most causal relationships
  - Low TPR: Miss many real causal effects

- **False Discovery Rate (FDR)**:
  $\text{FDR} = \frac{\text{FP}}{\text{FP} + \text{TP}}$ = fraction of predicted
  edges that are false
  - Low FDR: Predictions mostly correct
  - High FDR: Many false positives

- **F1 Score**:
  $\text{F1} = 2 \cdot \frac{\text{TPR} \cdot (1 - \text{FDR})}{\text{TPR} + (1 - \text{FDR})}$
  = harmonic mean balancing both

* Structure-Level and Complexity Metrics
- **Structural Hamming Distance**:
  $\text{SHD} = |\text{missing}| + |\text{extra}| + |\text{reversed}|$
  - Counts missing edges, false positives, reversed edge directions
  - Lower SHD is better (0 = perfect recovery)

- **Complexity metrics**: Number of Non-Zero edges (NNZ)

- **Best Practice**: Report TPR, FDR, F1, SHD together
  - Different stakeholders prioritize different metrics
  - Scientists want high TPR (find all mechanisms)
  - Practitioners want low FDR (avoid false claims)
  - Report NNZ to contextualize conservative vs liberal predictions

* Discovery Algorithm Families: Overview

- **Question**: _"How do we discover causal relationships from data?"_
  - Different data types and assumptions enable different approaches
  - No single algorithm works for all settings

- **Key Insight**: Causal discovery algorithms fall into distinct families based
  on their assumptions and data requirements
  - Constraint-based: Use conditional independence tests
  - Score-based: Optimize likelihood with regularization
  - Functional: Exploit non-linear or non-Gaussian properties
  - Interventional: Leverage experimental data from interventions

- **Trade-off**: More assumptions about the data $\to$ stronger causal
  guarantees but narrower applicability

* Discovery Algorithm Families: Detailed Comparison

\begingroup \scriptsize
| **Family**       | **Example**   | **Uses**                    | **Output** | **Key limit**                  |
| ---------------- | ------------- | --------------------------- | ---------- | ------------------------------ |
| Constraint-based | PC, FCI       | CI tests                    | CPDAG/PAG  | Sample size for CI tests       |
| Score-based      | GES, NOTEARS  | Likelihood + regularization | CPDAG / W  | Search / optimization hardness |
| Functional       | LiNGAM, ANM   | Non-Gaussianity, nonlinearity | Full DAG | Strong functional assumptions  |
| Interventional   | IGSP, JCI     | Data from interventions     | Full DAG   | Cost of experiments            |
\endgroup

- **Remark**: Output types differ in what the algorithm can identify
  - CPDAG/PAG: Partially directed acyclic graph (includes some undirected edges)
  - Full DAG: Complete directed acyclic graph (fully oriented graph)
  - Weight matrix $W$: Explicit edge strengths

* Constraint-Based Discovery

- **Intuition**: _"If two variables are independent given a set of others, they
  cannot have a direct causal link"_
  - Conditional independence (CI) is a fingerprint of the causal graph structure
  - Different graph structures imply different independence patterns

- **Working Principle**:
  - Start with a complete undirected graph (all variables connected)
  - Test conditional independence: are $X \perp Y | Z$?
  - Remove edges where CI holds
  - Orient edges using rules (e.g., detect colliders)

- **Example**: In a causal graph $X \to Z \to Y$, we expect $X \perp Y | Z$
  - If this CI holds empirically, we know $Z$ must be a mediator
  - If $X \perp Y | Z$ fails, $Z$ might not be the only path

- **Limitation**: Many different graphs can have the same CI pattern
  - Result is partially directed (undirected edges remain)

* Score-Based Discovery

- **Intuition**: _"The true causal graph should have high likelihood and be
  simple"_
  - Good models explain data well without overfitting
  - Regularization penalizes complexity

- **Working Principle**:
  - Define a score combining fit and complexity: Score$(G) = \text{Likelihood}(G) -
    \lambda \cdot \text{Complexity}(G)$
  - Search over possible DAGs to maximize the score
  - Use greedy algorithms (e.g., hill climbing) to avoid exponential search

- **Example**: GES (Greedy Equivalence Search)
  - Forward phase: Add edges that most improve score
  - Backward phase: Remove edges if they don't decrease score
  - Find a local optimum in the space of DAGs

- **Limitation**: Search is NP-hard; greedy algorithm finds local optima

* Functional Discovery

- **Intuition**: _"Causal mechanisms have special statistical properties that
  differ from confounding"_
  - True causal relationships exhibit asymmetry in the residuals
  - Fitting $Y = f(X) + \varepsilon$ works better than $X = f(Y) + \varepsilon'$

- **Working Principle** (example: LiNGAM):
  - Assume linear model: $X_i = \sum_j w_{ij} X_j + e_i$
  - Assume non-Gaussian noise: $e_i$ are independent and non-Gaussian
  - Non-Gaussianity breaks symmetry: fitting in reverse direction leaves
    dependent residuals
  - Identify the causal order using this asymmetry

- **Example**: $Y = 2X + e$ (causal) vs $X = 0.4Y + e'$ (spurious)
  - Non-Gaussian noise $e$ makes the forward model identifiable
  - Backward model would require dependent residuals $e'$

- **Limitation**: Strong assumptions (linear, non-Gaussian); breaks under
  Gaussianity

* Interventional Discovery

- **Intuition**: _"Intervening on a variable breaks confounding"_
  - Randomized experiments eliminate confounders by design
  - Multiple interventions can resolve ambiguities from observational data

- **Working Principle**:
  - Perform interventions: set variable $X$ to value $x$ and observe effects
  - Interventional data follows different distribution than observational
  - Combine with constraint-based or other methods on interventional data

- **Example**: Effect of price on sales
  - Observational: price and sales are correlated (confounded by demand)
  - Interventional: set price $= 5$ regardless of demand, observe sales
  - Causal effect is identified under randomization assumption

- **Advantage**: Can identify full DAG with sufficient interventions
  - Gold standard in experimental science
  - Limitation: Cost and feasibility of experiments

* Choosing an Algorithm Family

// TODO(gp): Maybe convert to table?

- **Constraint-based** (PC, FCI)
  - Use when: Observational data with large sample size
  - Advantage: Minimal distributional assumptions
  - Limitation: Many edges may remain undirected; high sample requirement

- **Score-based** (GES, NOTEARS)
  - Use when: Enough data to reliably estimate likelihood
  - Advantage: Flexible to different model specifications
  - Limitation: NP-hard search problem; local optima common

- **Functional** (LiNGAM, ANM)
  - Use when: Strong domain knowledge about the functional form
  - Advantage: Can identify full DAG from observational data
  - Limitation: Requires assumptions (linear, non-Gaussian; or additive noise)

- **Interventional** (IGSP, JCI)
  - Use when: Experimental data available from targeted interventions
  - Advantage: Strongest causal guarantees; can resolve ambiguities
  - Limitation: Expensive; requires designing good experiments

* Validating a Discovered DAG

- A recovered DAG is a *hypothesis* until validated

- **Validation strategies**:
  1. _Expert review_: do edges match known mechanisms?
  2. _Refutation tests_: does data contradict implied conditional independencies?
  3. _Sensitivity analysis_: do results change under perturbations of the
     assumptions?
  4. _Interventional check_: do targeted experiments confirm predicted effects?

* Refutation Testing

- **Definition**: given a discovered DAG $G$, check each implied independence
  against data

- **Procedure**:
  1. Enumerate CI statements implied by $G$ (via d-separation)
  2. Test each on held-out data using an appropriate CI test
  3. Count violations; low count $\Rightarrow$ DAG is _consistent_

- **Placebo tests**: introduce a known-null edge; discovery should
  assign it low weight

- **Example**
  - Replace treatment with a random variable; effect should vanish
  - Add an unobserved confounder; estimate sensitivity of conclusions

- **Remark**: if the conditional independence was used to discover the causal
  DAG, then a refutation test is redundant

- **Question**: what if the DAG passes all refutation tests?
  - **Answer**: it is _consistent_, not _proven_
  - Causality is never established by data alone; always requires domain-informed
    interpretation

// Papers:
// - 2017, Peters et al., "Elements of Causal Inference"
// - 2018, Squires et al., "Permutation-based causal structure learning with unknown interventions"

* When Discovery Should Change Your DAG

- **Update your DAG when**:
  - Multiple independent algorithms converge on the same edge
  - The new edge is _mechanistically plausible_: the discovered edge must be
    theoretically coherent with domain knowledge, not just statistically
    present
  - Refutation tests fail your current DAG
  - Interventional experiments confirm the discovered structure

- **Do not update your DAG when**:
  - Edge contradicts well-established mechanism (e.g., time order, physics)
  - Result hinges on weak faithfulness near independence boundary
  - Sample size is too small for reliable CI tests
  - Hidden confounders could explain the edge

- **Rule of thumb**: Discovery
  - Shifts the _prior probability_ of an edge
  - Does not provide a _posterior certainty_

* Causal Discovery: Key Takeaways

- **Causal discovery** infers DAG structure from data, but only under strong
  assumptions (Markov, faithfulness, sufficiency)

- **Identifiability is limited**
  - Observational data recover only the Markov equivalence class
  - Interventions or functional assumptions are needed for full identification

- **Three algorithm families**:
  - Constraint-based (PC, FCI)
  - Score-based (GES, NOTEARS)
  - Functional / non-Gaussian (LiNGAM)

- **Discovery $\neq$ truth**
  - Use it to _generate_ hypotheses
  - Combine with domain knowledge and interventions
  - Validate via refutation and expert review
  - Automated discovery is a powerful exploratory tool, not a replacement for
    scientific reasoning
  - Treat discovery as informative but fallible
